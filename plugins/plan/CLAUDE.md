# Plan Plugin — Operating Rules

- **Complete specs only.** Output is a feature request specification — never an implementation plan, architecture doc, or task list.
- **Evidence before claims.** Never assert something is "new" or "doesn't exist" without searching the codebase first.
- **Clarification is mandatory.** Do not finalize until all material ambiguity is resolved. No "Open Questions" in the output.
- **Fewer, better questions.** Batch questions. Prioritize ones that change the shape of the requirement. Apply trivial defaults silently.
- **Challenge wrong-problem framing.** Detect when the request is really a config change, doc update, or duplicate of existing capability.
- **No file modifications.** This plugin analyzes — it does not edit code. Final output goes to the conversation.
- **Decision ledger is source of truth.** Every claim in the output must trace back to the ledger.
